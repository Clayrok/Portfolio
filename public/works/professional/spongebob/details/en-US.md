# SpongeBob: Battle for Bikini Bottom – Rehydrated

## Project Overview
I contributed to the development of SpongeBob: Battle for Bikini Bottom – Rehydrated, focusing on the mobile adaptation of this classic title. My work centered on porting and optimizing the user interface to ensure a seamless experience on handheld devices.

## My Role: UI & Mobile Developer
Working within Unreal Engine 4, I bridged the gap between low-level C++ logic and designer-friendly Blueprints. My responsibilities included:
* **Mobile UI Refactoring:** Redesigning and implementing user interfaces specifically optimized for mobile screens and touch controls.
* **Optimization:** Improving performance and memory usage to meet mobile hardware constraints.
* **Bug Fixing:** Identifying and resolving technical issues unique to the mobile port.
* **Architecture:** Exposing C++ functionality to Blueprints selectively, ensuring a clean and efficient workflow for designers to manage UI interaction behaviors.

## Challenges & Solutions

### 1. Adaptation to Remote Work (COVID-19 Transition)
**Problem:** This project coincided with the first global lockdown. Transitioning to full-time remote work for the first time presented challenges in maintaining productivity and a healthy work-life balance.
**Solution:** I established a dedicated workspace in a separate room to physically and mentally isolate my professional environment from my personal life. This discipline allowed me to maintain high output and clear communication with the team throughout the confinement period.

### 2. Complex UI Layouts on Small Screens
**Problem:** The original game featured information-dense interfaces designed for large TVs and monitors, which were unreadable or cluttered on mobile displays.
**Solution:** I re-engineered the entire interface for mobile ergonomics. By implementing conditional rendering logic, I ensured that secondary information was tucked away in sub-menus, while essential gameplay data remained prominent and touch-friendly.

### 3. Hardware Performance Constraints
**Problem:** Initial builds suffered from low framerates and high memory usage on mid-range mobile devices due to the high-fidelity assets inherited from the console version.
**Solution:** I led a targeted optimization pass, focusing on asset compression and polygon simplification. By streamlining the UI textures and optimizing Blueprint execution paths, I significantly improved the game's performance without sacrificing visual clarity.
