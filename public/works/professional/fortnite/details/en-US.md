# Fortnite

## Project Overview
I contributed to the development and maintenance of Fortnite, working within the Unreal Engine ecosystem to enhance the player experience across multiple platforms. My work focused on UI development, feature implementation, and system optimization.

## My Role: UI & Gameplay Developer
Working with both C++ and Blueprints, I was responsible for delivering high-quality user interfaces and gameplay features, including:
* **UI Development:** Created and animated complex interfaces using the UMG (Unreal Motion Graphics) editor for the In-Game UI and the Item Shop.
* **Mobile Systems:** Developed mobile-specific features, including data download progress bars and system notifications.
* **Feature Expansion:** Integrated new logic into existing systems such as News Banners, Door Bashing mechanics, and Sound Visualizers.
* **Collaboration:** Designed flexible Blueprint node structures to allow designers and fellow developers to easily customize UI behaviors and animations.
* **Bug Fixing:** Identified and resolved critical gameplay and interface bugs to ensure a polished user experience.

## Challenges & Solutions

### 1. Parallel Development and Backend Dependencies
**Problem:** I frequently needed to build complex UI systems before the corresponding backend functionality was fully developed, leading to a "blocked" workflow.
**Solution:** I initiated regular cross-departmental syncs with the backend team to define and agree upon a list of "backend hooks" and API structures in advance. This allowed me to build the frontend logic using these stubs, ensuring seamless integration once the backend code was pushed.

### 2. Legacy Inheritance Issues in Thousands of Assets
**Problem:** The project contained thousands of Blueprints with outdated or incorrect parent class assignments, making global updates or optimizations nearly impossible to manage manually.
**Solution:** I developed an automation script that allowed for the batch-reparenting of selected assets. This significantly reduced manual labor and eliminated the human error associated with updating the inheritance hierarchy across such a vast codebase.

### 3. Synchronizing with a Massive Global Workforce
**Problem:** Working in a studio environment with thousands of employees spread across different time zones made it difficult to maintain a unified direction and avoid redundant work.
**Solution:** I helped establish and maintain shared digital boards and summary documentation for our sub-groups. By participating in multiple daily "sync" meetings, I ensured our team remained aligned with the global roadmap and that all technical dependencies were synchronized in real-time.
